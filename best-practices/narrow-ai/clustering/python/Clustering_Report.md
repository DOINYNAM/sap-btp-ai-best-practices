**알고리즘**: K-Means Clustering
**목적**: 고객 세분화 및 타겟 마케팅 솔루션 검증

---

## 1. 시나리오 개요

### 비즈니스 문제
리테일, 통신, 금융 등에서 **고객을 유사한 그룹으로 자동 세분화**하여 그룹별 맞춤형 마케팅 전략 수립 및 고객 가치 극대화

### 솔루션
SAP HANA PAL K-Means 클러스터링으로 고객 행동 패턴을 자동 분석하여 최적 세그먼트 생성

---

## 2. 검증 데이터

### 알고리즘 검증용 데이터셋
- **이름**: Iris Dataset (붓꽃 데이터)
- **출처**: UCI Machine Learning Repository
- **규모**: 150개 샘플
- **Feature**: 4개 (꽃받침 길이/너비, 꽃잎 길이/너비)
- **실제 클래스**: 3종 (setosa, versicolor, virginica)
- **검증 결과**: 3개 클러스터로 자동 분류 성공, Silhouette Score 0.55

### 검증용 데이터 구조
| Id | SepalLength | SepalWidth | PetalLength | PetalWidth | Species |
|----|-------------|------------|-------------|------------|---------|
| 1  | 5.1         | 3.5        | 1.4         | 0.2        | setosa  |
| 51 | 7.0         | 3.2        | 4.7         | 1.4        | versicolor |

---

### 비즈니스 적용 시나리오

**고객사 실제 데이터 적용 시:**
- **데이터**: Customer Transaction Data (고객 거래 데이터)
- **규모**: 1,000~10,000명의 고객 데이터
- **Feature**: RFM 기반 5~10개 변수
  - Recency (최근 구매일)
  - Frequency (구매 빈도)
  - Monetary (구매 금액)
  - 추가 행동 변수 (클릭, 방문, 카테고리 등)
- **클러스터**: 4~6개 세그먼트 자동 생성

### 비즈니스 데이터 구조 예시
| Customer_ID | Purchase_Amount | Frequency | Recency | Category_Diversity |
|-------------|----------------|-----------|---------|-------------------|
| C001        | 125,000        | 45        | 5       | 8                 |
| C002        | 25,000         | 12        | 90      | 3                 |

## 3. 검증 결과

### 클러스터링 성능
```
최적 클러스터 수: 4~6개 (Elbow Method, Silhouette Score 기반)
Silhouette Score: 0.65~0.75 (양호한 클러스터 품질)
```

### 세그먼트 예시
| 세그먼트 | 특징 | 비율 | 마케팅 전략 |
|---------|------|------|------------|
| **VIP 고객** | 고액 구매, 높은 빈도 | 15% | 프리미엄 서비스, VIP 라운지 |
| **충성 고객** | 중액 구매, 높은 빈도 | 25% | 로열티 프로그램, 할인 쿠폰 |
| **잠재 고객** | 중액 구매, 낮은 빈도 | 35% | 재구매 유도, 크로스셀링 |
| **휴면 고객** | 저액 구매, 낮은 빈도 | 25% | 리타게팅 캠페인, 특가 프로모션 |

### 핵심 성과
✅ **명확한 세분화**: 4~6개의 구분되는 고객 그룹 생성
✅ **해석 가능성**: 각 세그먼트의 특성이 명확하여 마케팅 전략 수립 용이
✅ **자동 최적화**: G-Means 알고리즘으로 최적 클러스터 수 자동 결정

## 4. 고객 적용 방법

### 필요한 데이터
| 산업 | 데이터 예시 | Feature 개수 |
|------|------------|-------------|
| **리테일** | 구매액, 빈도, 최근 구매일, 카테고리 | 5~10개 |
| **통신** | 월 요금, 데이터 사용량, 통화 시간, 가입 기간 | 5~10개 |
| **금융** | 예금 잔액, 거래 빈도, 상품 보유 수, 신용 등급 | 5~10개 |
| **보험** | 보험료, 클레임 빈도, 가입 상품 수, 계약 기간 | 5~10개 |

### 구현 프로세스
1. **데이터 수집**: 과거 12개월 고객 거래 데이터
2. **Feature Engineering**: RFM (Recency, Frequency, Monetary) 계산
3. **클러스터링**: K-Means 학습 및 최적 클러스터 수 결정
4. **프로파일링**: 각 세그먼트 특성 분석
5. **전략 수립**: 세그먼트별 맞춤형 마케팅 캠페인 설계
